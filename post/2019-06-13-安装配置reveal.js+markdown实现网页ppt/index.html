安装配置 reveal.js+Markdown 实现网页 ppt
配置 reveal.js，对 css 模板进行微调，使用 external Markdown
2019-06-13 15:06 label Markdown   reveal.js  

关于PPT

因为之前从事管理战略咨询的缘故，PPT 是必备技能。thinkcell+键盘快捷键玩到飞起，巅峰时刻一晚大几十页PPT没有任何问题 - 于是脑海中非常根深蒂固的印象就是 PPT = Microsoft Powerpoint，最好的 PPT 模板就是麦肯锡的一套。过去三四年转行互联网，这样的认知逐渐被打破，尤其在以下方面，看到了很多 PowerPoint 没法解决（或不适应）的问题：

  1. 非线性逻辑结构：PPT 的翻页是线性且单维的，这让我现在很多时候用脑图而不是 PPT 来思考和陈述
  2. 对于包括代码、视频、公式等形式的支持落后：记得自己2006年开始用 Linux（Ubuntu），就是因为写论文的时候不能忍受输入数学公式的繁琐，转而用 vim + LaTex 生成 PDF。时至今日，这些问题依然在 office suite 里面没有得到特别好的解决
  3. 轻便：如果不是之前管理战略咨询那样的 PPT（极其严谨的图文排布、12-16号字打便天下，高密度信息结构），其实是不需要 PowerPoint 那么重的工具的。同时，过去三四年我自己的管理决策一直是在做减法，因此在最终思路的呈现上反而强调 KISS 原则，不像之前呈现给500强 CXO 或者 PE 投资人那么需要信息丰富度和高级逻辑能力，转而力求“把事情用最简单的方式说清楚，并且让绝大多数听众第一时间就能明白吸收”。在这种情况下，PowerPoint 略显沉闷和笨重
  4. 云和协作：这点 Google Suite 比 Office Suite 好很多。当然我不觉得 Google Suite 总体产品做的很好，只是会拿 Google Doc 和 Google Keep 来记录日常思考，最近有被 Markdown 取代的迹象。实时云存储一定是趋势，PowerPoint 动作太慢了，背后其实是有隐性风险的，比如我相信很多人都经历过本机保存不及时活白干，或者团队内部搞 PPT 版本控制问题等。

当然，PowerPoint 的优势依然明显，比如任何时候，我需要准备一份最严谨的商业陈述，或者 BD/ 销售材料，那么 PowerPoint 仍然是我的最优解。但在其他时刻，比如团队内部分享，或者一些产品讨论等方面，我在过去一段时间里的确在认真考虑使用更加灵活和轻量级的其他工具。

为什么是 Markdown + reveal.js

  1. 还是尝试阶段，可能有更好的工具
  2. 过去一周使用 Markdown 的体验非常愉快。Markdown 比当年写 LaTex 文档入门门槛低太多了，而且排版还是很舒服的
  3. 个人是所想即所得的忠实拥护者，我对工具的要求是在聚焦写完 Markdown 文档以后，能否最低成本地以我想要的方式“逐页”展现出来
  4. Reveal.js的社区文档做的不错，一开始的demo还是很吸引人的。粗略看了一圈，也不复杂，就先试起来了

Reveal.js的配置

  • 基本 follow GitHub 上的官方教程，我采用的是 Full Setup，就是装了一个比较完整的 node.js + npm 环境在机子上
  • 不喜欢用 “internal” Markdown 的方式来写，每一页还要前后加 <section> </section> 也太烦人了，直接采用 External Markdown 的方式来处理比较好，最快上手的方式是直接在安装目录下的 index.html 上面修改保存，省的一开始配置
  • 一个关键定义是 Markdown 文档里面怎么分页，官方给的方式是横向空三行、纵向空两行

data-separator="^\n\n\n”
data-separator-vertical="^\n\n”

我采用的方式是横向空一行，加三个-，再空一行;纵向纵向空一行，加两个-，再空一行。纯粹觉得这样比较安全

data-separator="^\n—\n$”
data-separator-vertical="^\n–\n$”

  • 默认的字体太大了，我偷懒就改了一下 css 里面 theme 文件的 font-size，从原来40改成24
.reveal {
font-family: "Source Sans Pro", Helvetica, sans-serif;
font-size: 24px;
font-weight: normal;
color: #fff; }

基本能用了。

待解决问题

  • 表格的宽度设定，跟用 Markdown 之间生成样子不一样
  • 插入图片和视频
  • 页面里一些元素的调整，比如引用，现在的版面占用比较大

2019.6.19 更新

插入图片选择了 PicGo + github 作为图床，每次上传完照片之后，进入 PicGo 的相片区，选择复制，可以自动生成 Markdown 文档的图片链接，粘贴进文档完事儿。

插入视频选择了用 iframe 方式，具体见 190619-reveal-md-格式模版
